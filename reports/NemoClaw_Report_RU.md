# NVIDIA NemoClaw — Детальный отчёт для руководства и службы безопасности

## 1. Краткое описание

**NVIDIA NemoClaw** — open-source reference-стек (лицензия **Apache 2.0**), обеспечивающий безопасный запуск AI-агентов (OpenClaw) внутри NVIDIA OpenShell — изолированной среды исполнения с многоуровневой защитой. Проект создан в марте 2026, текущий статус — **alpha**.

**Ключевая идея:** NemoClaw оборачивает OpenClaw (аналог Claude Code) корпоративным слоем безопасности: песочница с Landlock/seccomp/netns, контроль сетевого трафика, маршрутизация инференса через шлюз, аудит действий агента.

### Жизненный цикл Blueprint

1. **Resolve** — определение версии артефакта, проверка совместимости
2. **Verify** — проверка дайджеста артефакта
3. **Plan** — определение ресурсов OpenShell (gateway, sandbox, policy, inference route)
4. **Apply** — выполнение плана через CLI OpenShell
5. **Status** — отчёт о текущем состоянии

## 4. Ключевые механизмы безопасности

### 4.1 Песочница (Sandbox Isolation)

| Уровень | Технология | Защита |
|---------|-----------|--------|
| 1 | **Docker-контейнер** | Изоляция процессов, файловой системы |
| 2 | **Landlock LSM** | Ограничение доступа к файлам (read-only для /usr, /lib, /etc) |
| 3 | **seccomp** | Фильтрация системных вызовов |
| 4 | **Network namespaces (netns)** | Сетевая изоляция |

### 4.2 Файловая система

| Путь | Доступ |
|------|--------|
| `/sandbox`, `/tmp`, `/dev/null` | Чтение + запись |
| `/usr`, `/lib`, `/proc`, `/etc`, `/app`, `/var/log` | Только чтение |
| Все остальные пути | Запрещены |

Процесс в песочнице запускается от выделенного пользователя `sandbox`.

### 4.3 Сетевой контроль

- **Strict-by-default**: весь трафик запрещён, кроме явно разрешённых эндпоинтов
- **Базовая политика** определена в `openclaw-sandbox.yaml`
- **Предодобренные эндпоинты**: NVIDIA API, GitHub, npm registry, Telegram API
- **Inference через gateway**: агент не имеет прямого доступа к inference-провайдерам

### 4.4 Operator Approval Flow

Когда агент пытается обратиться к неизвестному хосту:

1. OpenShell перехватывает соединение и блокирует его
2. Запрос отображается в TUI-интерфейсе оператора
3. Оператор одобряет или отклоняет в реальном времени
4. Одобренный эндпоинт добавляется в политику сессии

### 4.5 Верификация артефактов

- Plugin проверяет дайджест Blueprint-артефакта перед исполнением
- Проверка совместимости версий (min_openshell_version, min_openclaw_version)

## 6. Возможности и интеграции

### 6.1 Основные возможности

- Безопасный запуск AI-агентов в изолированных контейнерах
- Многоуровневая защита (4 уровня изоляции)
- Контроль сетевого трафика с операторским одобрением
- Маршрутизация инференса через защищённый шлюз
- Поддержка нескольких inference-провайдеров
- Аудит и логирование всех действий агента
- CLI-интерфейс для управления

### 6.2 Поддерживаемые модели/провайдеры инференса

| Провайдер | Описание |
|-----------|----------|
| **NVIDIA Cloud API** | Nemotron и другие модели NVIDIA (по умолчанию) |
| **Local NIM** | NVIDIA Inference Microservice на локальном сервере |
| **vLLM** | Open-source inference server на localhost |
| **Ollama** | Локальные модели (LLaMA, Mistral, Qwen и др.) |

### 6.3 CLI-команды

| Команда | Назначение |
|---------|-----------|
| `nemoclaw launch` | Установка и запуск песочницы |
| `nemoclaw connect` | Подключение к песочнице (shell) |
| `nemoclaw status` | Состояние blueprint и sandbox |
| `nemoclaw logs` | Просмотр логов |
| `nemoclaw chat` | Интерактивный чат с агентом |

## 7. Преимущества для корпоративного использования

1. **Многоуровневая изоляция**: 4 слоя защиты предотвращают утечку данных и несанкционированный доступ
2. **Контроль сетевого трафика**: оператор контролирует каждый исходящий запрос агента
3. **Локальный инференс**: возможность полной работы без отправки данных во внешние облака (vLLM, Ollama, NIM)
4. **Аудит**: полный трейс действий агента
5. **Open source (Apache 2.0)**: нет vendor lock-in, возможность модификации под требования организации
6. **NVIDIA backing**: поддержка от NVIDIA гарантирует развитие проекта

## 10. Рекомендации

1. **Для пилотного проекта**: развернуть NemoClaw + Ollama + MiniMax M2.7 / Qwen3 на тестовом сервере
2. **Для production**: NemoClaw + vLLM + Nemotron/Qwen3-Coder на GPU-сервере
